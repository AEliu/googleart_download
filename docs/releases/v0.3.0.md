# v0.3.0

Third public release of `googleart-download`.

## Highlights

- Add `--tile-only` so users can download artwork tiles into a visible `.tiles` directory without stitching a final image.
- Add `--stitch-from-tiles` so users can later assemble a final image from an existing `.tiles` directory.
- Make tile-only cache identity explicit so different artworks do not accidentally reuse the same visible tile directory.
- Close tile-only `--output-conflict skip` semantics so complete existing tile directories are reported as skipped instead of being treated like fresh successful downloads.
- Expand user documentation for tile-only outputs and stitch-from-tiles workflows in both English and Chinese docs.

## Notes

- `--stitch-from-tiles` currently writes only the final stitched image. It does not yet restore EXIF or JSON sidecar metadata from the earlier tile-only run.
- Tile-only outputs store `tiles/*.tile` and `state.json` in a visible `.tiles` directory.
- Very large artworks still follow the safer TIFF/BigTIFF path by default when the selected stitch backend requires it.
